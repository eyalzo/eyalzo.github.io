# פרומפט להמרת הצעות חוק פעילות (Active Laws) ל-JSON

מצורף קובץ אקסל בשם: `2026-06-14_law_proposals_k21_to_k25.xlsx`.
הקובץ מכיל נתונים על הצעות חוק. אנא קרא את הנתונים והמר אותם למערך של אובייקטים בפורמט JSON, המיועד להשתלב תחת המפתח `items` במערכת שלנו.

**הנחיות מבנה מחמירות:**
1. סוג הפריט (`type`) חייב להיות תמיד: `"activeLaw"`.
2. **אין** ליצור שדה `source` עבור פריטים אלו.
3. תאריכים יש להמיר לשדה `date` אחיד בלבד (בפורמט "DD/MM/YYYY" או "YYYY-MM-DD"). אין להשתמש בשדות נפרדים של יום, חודש ושנה.
4. השדה `impact` יוצג רק אם קיים עבורו מידע.

**מיפוי השדות מהאקסל למפתחות ב-JSON:**
* **date**: תאריך ההצעה (לקחת מעמודת: [שם עמודת התאריך באקסל]).
* **relevance**: ציון קשר מספרי בין 1 ל-10 (לקחת מעמודת: [שם עמודת ציון הקשר]).
* **type**: מחרוזת קבועה - `"activeLaw"`.
* **idCode**: מזהה ההצעה הייחודי, למשל פ/1234/25 (לקחת מעמודת: [שם עמודת המזהה]).
* **link**: קישור מלא למסמך, חובה שיתחיל ב-http/https (לקחת מעמודת: [שם עמודת הקישור]).
* **author**: יוזם החוק / חברי הכנסת (לקחת מעמודת: [שם עמודת היוזמים]).
* **title**: שם הצעת החוק (לקחת מעמודת: [שם עמודת כותרת החוק]).
* **summary**: תקציר ההצעה (לקחת מעמודת: [שם עמודת התקציר]).
* **impact**: השלכה מעשית (לקחת מעמודת: [שם עמודת ההשפעה]).
* **keywords**: מערך של מחרוזות, 3-4 מילות מפתח (לקחת מעמודת: [שם עמודת מילות מפתח]).
* **activeLawStatus**: הסטטוס הנוכחי, למשל "הכנה לקריאה ראשונה" (לקחת מעמודת: [שם עמודת סטטוס]).
* **activeLawType**: סוג ההצעה, למשל "פרטית" או "ממשלתית" (לקחת מעמודת: [שם עמודת סוג הצעה]).
* **activeLawVaada**: הוועדה המטפלת. אם אין, להשאיר מחרוזת ריקה (לקחת מעמודת: [שם עמודת ועדה מטפלת]).

אנא הפק רק את קוד ה-JSON התקין, מוכן להעתקה.
