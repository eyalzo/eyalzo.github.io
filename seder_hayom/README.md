# פרומפט להמרת מסמכי ממ"מ (MMM) ל-JSON בהקשר של `TOPIC_NAME`

מצורף קובץ נתונים (למשל `2026-06-14_mmm_docs.csv`) המכיל דוחות ומסמכים של מרכז המחקר והמידע של הכנסת (ממ"מ). 
אנא סרוק את המסמכים, מצא את המסמכים הרלוונטיים ביותר ל`TOPIC_NAME` (במיוחד לאור ההגדרה: `TOPIC_DESC`), והמר אותם למערך של אובייקטים בפורמט JSON, המיועד להשתלב תחת המפתח `items` במערכת שלנו.

**הנחיות מבנה מחמירות:**
1. סוג הפריט (`type`) חייב להיות תמיד: `"mmm"`.
2. **אין** ליצור שדה `source` עבור פריטים אלו.
3. תאריכים יש להמיר לשדה `date` אחיד בלבד (בפורמט "DD/MM/YYYY" או "YYYY-MM-DD").
4. יש לחלץ מתוך הקישור את פרמטר ה-`rid` ולייצר שדה `idCard` בפורמט: `"mmm_[rid_value]"`.
5. השדה `impact` חייב להיכתב בהלימה ישירה ל-`TOPIC_NAME` ולתאר כיצד המסמך שירת את הוועדות או את מקבלי ההחלטות בהקשר זה.

**מיפוי השדות מהקובץ למפתחות ב-JSON:**
* **date**: תאריך המסמך (לקחת מעמודת: [שם עמודת התאריך, למשל `date`]).
* **relevance**: ציון קשר מספרי בין 1 ל-10 המעיד על רלוונטיות ל`TOPIC_NAME`.
* **type**: מחרוזת קבועה - `"mmm"`.
* **link**: קישור מלא למסמך (לקחת מעמודת: [שם עמודת הקישור, למשל `link`]).
* **title**: כותרת המסמך (לקחת מעמודת: [שם עמודת הכותרת, למשל `title`]).
* **summary**: תקציר המסמך (לקחת מעמודת: [שם עמודת התקציר, למשל `summary`]).
* **impact**: השלכה מעשית וזיקה ל`TOPIC_NAME` (יש לנסח תיאור בהתבסס על התקציר וההקשר).
* **mmmRequester**: שם המבקש (לקחת מעמודת: [שם עמודת המבקש, למשל `requested_by`]). אם חסר, ציין "לא צוין".
* **mmmCategory**: סוג המסמך, למשל "סקירה" או "נתונים" (לקחת מעמודת: [שם עמודת הקטגוריה, למשל `category`]). אם חסר, ציין "לא צוין".
* **idCard**: מזהה מותאם המורכב מהקידומת `mmm_` בתוספת מספר ה-rid (מתוך הקישור).

אנא הפק רק את קוד ה-JSON התקין, מוכן להעתקה.
