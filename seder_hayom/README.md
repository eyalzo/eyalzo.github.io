# פרומפט להמרת מסמכי ממ"מ (MMM) ל-JSON בהקשר של `TOPIC_NAME`

מצורף קובץ נתונים (למשל `2026-06-14_mmm_docs.csv`) המכיל דוחות ומסמכים של מרכז המחקר והמידע של הכנסת (ממ"מ). 
העמודות המצופות בקובץ זה הן:
`id, date, title, author, additional_authors, teamleader, requested_by, additional_requesters, category, summary, link, keywords`

אנא סרוק את המסמכים, מצא את המסמכים הרלוונטיים ביותר ל-`TOPIC_NAME` (במיוחד לאור ההגדרה: `TOPIC_DESC`), והמר אותם לפורמט JSON.

**מכיוון שזוהי המשימה הראשונה, עליך לבנות את קובץ ה-JSON מההתחלה ממש.** צור את המעטפת הבאה, והכנס את המסמכים שיצרת לתוך המערך `documents` שתחת `mmmSection`:

```json
{
  "meta": {
    "title": "[כאן יכנס TOPIC_NAME]",
    "subtitle": "[כאן יכנס TOPIC_DESC אחרי ליטוש קל אם צריך]",
    "version": "1.0.0",
    "lastUpdate": "[הזן את תאריך יצירת הקובץ, למשל: 14/07/2026]"
  },
  "appConfiguration": {
    "mmmSection": {
      "documents": [
        // כאן יוזרקו האובייקטים של מסמכי הממ"מ שסיננת ועיבדת
      ]
    },
    "items": []
  }
}
```

**הנחיות מבנה מחמירות למסמכים:**
1. סוג הפריט (`type`) חייב להיות תמיד: `"mmm"`.
2. **אין** ליצור שדה `source` עבור פריטים אלו.
3. תאריכים יש להמיר לשדה `date` אחיד בלבד (בפורמט "DD/MM/YYYY" או "YYYY-MM-DD").
4. יש לחלץ מתוך הקישור את פרמטר ה-`rid` ולייצר שדה `idCode` בפורמט: `"mmm_[rid_value]"`.

**מיפוי השדות (המדויק) מהקובץ למפתחות ב-JSON:**
* **date**: תאריך המסמך (מתוך עמודת `date`).
* **relevance**: ציון קשר מספרי בין 1 ל-10 (הערכה וחיבור עצמאי שלך בהתבסס על `TOPIC_DESC`).
* **type**: מחרוזת קבועה - `"mmm"`.
* **link**: קישור מלא למסמך (מתוך עמודת `link`).
* **title**: כותרת המסמך (מתוך עמודת `title`).
* **summary**: תקציר המסמך (מתוך עמודת `summary`).
* **mmmRequester**: שם המבקש (מתוך עמודת `requested_by`). אם חסר, ציין "לא צוין".
* **mmmCategory**: סוג המסמך (מתוך עמודת `category`). אם חסר, ציין "לא צוין".
* **idCode**: כאמור, מזהה מותאם המורכב מהקידומת `mmm_` בתוספת מספר ה-rid שיש לחלץ מהקישור.

אנא הפק רק את קוד ה-JSON התקין (כולל המעטפת), מוכן להעתקה.

# פרומפט לכתיבת תקציר מנהלים (Executive Summary) ומדור "מה התחדש"

**שימו לב:** שלב זה יש לבצע **אך ורק לאחר** שכל רשימת המסמכים (מערכי ה-`items` ו-`documents`) הוכנה והושלמה במלואה.

**אני אספק לך כעת את קובץ ה-JSON המלא הכולל את כל המסמכים שאספנו עד כה.**
עליך לכתוב מחדש את תקציר המנהלים ואת מדור "מה התחדש" על סמך המסמכים שקיימים כעת ב-JSON. **אם כבר קיים אובייקט `execSummary` בקובץ, עליך לדרוס אותו לחלוטין ולעדכן אותו במידע החדש.** 
בנוסף, עליך לעדכן את השדות `version` (להעלות גרסה, למשל מ-1.0.0 ל-1.1.0) ו-`lastUpdate` (תאריך של היום) תחת אובייקט ה-`meta`.

**1. תקציר מנהלים אינטראקטיבי (`execSummary.text` ו-`execSummary.textRefs`):**
* עליך לכתוב תקציר המשקף את תמונת המצב הכוללת והעדכנית בנושא `TOPIC_NAME`. התכלול חייב להתבסס באופן שיטתי על המסמכים שאספת.
* **נפח מצופה:** התקציר צריך להיות ממוקד ותמציתי – כ-4 עד 6 משפטים בסך הכל (סביב 100-150 מילים).
* **מבנה הטקסט:** התקציר חייב להיות מורכב ממשפטים קצרים וברורים המופרדים בנקודה ורווח (`. `). שיבוץ הטקסט ייעשה לשדה המדויק `execSummary.text`.
* **קישור למסמכים (רפרנסים - Modal):** המערכת הופכת כל רפרנס לכפתור ממוספר (1, 2, 3) המופיע בסוף המשפט ופותח חלון עם פרטי המסמך. 
  * כל אובייקט במערך `textRefs` יכיל `sentenceIndex` (מיקום המשפט בתקציר, מתחיל ב-1) ו-`refId` (מזהה ה-`idCode` של המסמך).
  * אם יש למשפט אחד מספר מקורות, צור עבורו מספר אובייקטים ב-`textRefs` עם אותו `sentenceIndex`. 

**2. מדור "מה התחדש" (`whatsNewItems`):**
* סרוק את כל המסמכים ב-JSON ושלף את העדכניים והחשובים ביותר, תחת הכללים הבאים:
* **תנאי 1 (כמות ומסגרת זמן):** עליך לבחור **בין 3 ל-5 פריטים הכי חשובים**. הפריטים חייבים להיות אך ורק מ**חצי השנה האחרונה**.
* **תנאי 2 (גיוון):** השתדל לגוון ככל הניתן – עדיף לבחור פריט 1 מכל קטגוריה (למשל: הצעת חוק אחת, דיון ועדה אחד, מסמך ממ"מ אחד).
* **מבנה הפריט:** כל פריט בתוך המערך `whatsNewItems` יכיל מחרוזת `text` (משפט מתמצת של מה שקרה/פורסם) ושדה `refId` עם המזהה התואם למסמך.

אנא הפק והחזר לי את קובץ ה-JSON השלם והסופי, כשהוא כולל את עדכוני ה-`meta`, את מערכי המסמכים, ואת ה-`execSummary` החדש והמעודכן.

# פרומפט ליצירת קשרים בין מסמכים (Related Documents)

**שימו לב - תנאי מקדים קריטי:** שלב זה יש לבצע **אך ורק לאחר** שכל רשימת המסמכים (מערך ה-`items`) הוכנה במלואה, נבדקה, ולכל מסמך הוקצה מזהה ייחודי, תקין וסופי (`idCode`). 

פעולה זו שואבת את הנתונים מתוך קובץ ה-JSON המלא והמוכן, מנתחת את ההקשרים התוכניים בין המסמכים השונים, ומעדכנת את אותו קובץ JSON (מוסיפה את שדה הקשרים לפריטים הרלוונטיים).

**הנחיות לזיהוי ויצירת קשרים:**
1. עליך (ה-AI) לסרוק את כל המסמכים בקובץ ה-JSON הקיים ולזהות קשרים ענייניים הדוקים. למשל: דוח מבקר המדינה שהוביל לדיון ועדה, הצעת חוק שנולדה בעקבות מסמך ממ"מ, או רצף דיונים באותו נושא מרכזי.
2. עבור כל מסמך שיש לו זיקה ברורה למסמך אחר במאגר, יש להוסיף את השדה `relatedDocs` למסמך (בדרך כלל למסמך המאוחר יותר כרונולוגית שמפנה לישן יותר).
3. השדה `relatedDocs` יכיל מערך של אובייקטים. כל אובייקט חייב להכיל את המבנה הבא:
   * `idCode`: המזהה המדויק של המסמך המקושר (חייב להיות תואם ב-100% ל-`idCode` במערך המסמכים).
   * `label`: מחרוזת קצרה (עד 4-5 מילים) המתארת את סוג הקשר (למשל: "דוח מבקר בנושא פרוטקשן", "הצעת חוק בעקבות הדיון").
   * `weight`: עוצמת הקשר במספר בין 1 ל-10 (כאשר 10 הוא קשר ישיר ובלתי נפרד).
4. **הדדיות (Bidirectional):** מספיק להוסיף את ההקשר רק באחד מהמסמכים. אין צורך לייצר כפילויות ולרשום את הקשר גם במסמך המקור וגם במסמך היעד, כיוון שמערכת התצוגה יודעת להשלים את רשת הקשרים באופן אוטומטי ולהציג את המסמך אצל כולם.

**דוגמה למבנה השדה שיש להוסיף בתוך אובייקט המסמך:**
```json
"relatedDocs": [
  {
    "idCode": "דוח 74ג",
    "label": "דוח מבקר בנושא הדיון",
    "weight": 9
  },
  {
    "idCode": "פ/6766/25",
    "label": "הצעת חוק מקבילה",
    "weight": 8
  }
]
```

# הנחיות מבנה להזרקת טאבים וקישורים חיצוניים (`appConfiguration`)

המערכת פועלת כעת בממשק טאבים (SPA). ניתן להזריק תוכן חיצוני ישירות כטאבים חדשים:

**1. הזרקת מקורות מידע מלאים כטאב (`externalFrames`):**
אם ברצונך להטמיע דשבורד חיצוני (למשל PowerBI או מפת GIS) בתוך iframe, צור אובייקט תחת `externalFrames`:
* **title**: שם הטאב שיופיע למעלה. יקבל אוטומטית אייקון 🌐.
* **url**: הקישור למערכת החיצונית (חובה שיתמוך בהטמעת iframe).
* **tab**: (אופציונלי) מספרו הסידורי של הטאב בסרגל הניווט העליון. למשל, `"tab": 2` ידחוף אותו להיות הטאב השני. אם לא יסופק מספר, הוא יוצב כטאב האחרון בשורה.
